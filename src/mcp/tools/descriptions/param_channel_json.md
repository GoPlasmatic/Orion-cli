JSON string of the channel definition. Required fields: "name" (string), "channel_type" ("sync" or "async"), "protocol" ("http", "rest", or "kafka"), "workflow_id" (ID of the workflow to execute). Optional: "description", "methods" (array of HTTP methods, e.g. ["GET", "POST"]), "route_pattern" (REST route pattern, e.g. "/orders/{id}/details"), "topic" (Kafka topic name), "consumer_group" (Kafka consumer group), "priority" (higher = matched first, default: 0), "config" (per-channel configuration object with optional fields: "rate_limit", "timeout_ms", "cors", "backpressure", "input_validation"). Example: {"name": "orders", "channel_type": "sync", "protocol": "rest", "methods": ["GET", "POST"], "route_pattern": "/orders/{id}", "workflow_id": "process-orders", "config": {"timeout_ms": 5000}}
