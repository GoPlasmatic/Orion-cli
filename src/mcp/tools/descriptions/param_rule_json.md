JSON string of the rule definition. Required fields: "name" (string), "tasks" (array of task objects). Optional: "channel" (default: "default"), "priority" (higher runs first, default: 0), "condition" (JSONLogic, default: true), "description", "tags", "continue_on_error". Each task: {"id": "...", "name": "...", "function": {"name": "parse_json|parse_xml|map|filter|validation|http_call|log|publish_json|publish_xml|publish_kafka", "input": {...}}}. Start most rules with a parse_json task: {"name": "parse_json", "input": {"source": "payload", "target": "order"}}. Validation uses rules array: {"name": "validation", "input": {"rules": [{"logic": <JSONLogic>, "message": "..."}]}}. publish_json uses source/target: {"name": "publish_json", "input": {"source": "order", "target": "json_string"}}. http_call requires connector: {"name": "http_call", "input": {"connector": "name", "method": "POST", "path": "/api/...", "body_logic": ..., "response_path": "data.result"}}. See the rules_create tool description for full format details and examples.
